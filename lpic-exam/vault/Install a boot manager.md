**Grub** is the boot loader used on most Linux distributions. It can load the Linux kernel or other operating systems, such as Windows, and can handle multiple kernel images and parameters.


## Installing GRUB 2

1. Check whether there is a boot partition on the disk.

```bash
fdisk -l /dev/sda
```

or `lsblk`

2. Create a temporary directory under `/mnt` and mount the partition under it:

```bash
mkdir /mnt/tmp
mount /dev/sda2 /mnt/tmp
```

3. The run `grub-install`, pointing it to the boot device and the directory where boot partition is mounted.

```bash
grub-install --boot-directory=/mnt/tmp /dev/sda2
```

or

✅ If you are installing to a system which does not have a boot partition, but just a `/boot` directory on the root filesystem, point `grub-install` to it.

```bash
grub-install --boot-directory=/boot /dev/sda2
```

## Configuring GRUB 2

The default configuration file for GRUB 2 is `/boot/grub/grub.cfg`. This file is automatically generated and manual editing is not recommended. To make changes to the GRUB configuration, you need to edit the file `/etc/default/grub` and then run the `update-grub` utility to generate a compliant file.

| Num. | Directory             | Purpose                                            |
|:----:| --------------------- | -------------------------------------------------- |
|  1   | `/boot/grub/grub.cfg` | The default configuration file for GRUB 2          |
|  2   | `/etc/default/grub`   | In order to make changes to the GRUB use this file |

| Command                                | Purpose                                                                             |
| -------------------------------------- | ----------------------------------------------------------------------------------- |
| `grub-install`                         | Install bootloader files.                                                           |
| `update-grub`                          | Run this command to generate a compliant file after modifying file in directory (1) |
| `grub-mkconfig -o /boot/grub/grub.cfg` | The command `update-grub` is a shortcut for this one.                               |

The most important options in the file `/etc/default/grub` to control the behavior of GRUB 2

| Option                         | Purpose                                                                                                                                                                                                                       |
| ------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **GRUB_DEFAULT**               | The default menu entry to boot                                                                                                                                                                                                |
| **GRUB_SAVEDEFAULTE**          | If this option is set to `true` and `GRUB_DEFAULT` is set to saved, then the default boot option will always be the last one selected in the boot menu.                                                                       |
| **GRUB_TIMEOUT**               | The timeout, in seconds, before the default menu entry is selected. If set to `0` system default option. If set to `-1` user select option.                                                                                   |
| **GRUB_CMDLINE_LINUX**         | This lists command line options that will be added to entries for the Linux kernel.                                                                                                                                           |
| **GRUB_CMDLINE_LINUX_DEFAULT** | One menu with the default options and one entry for recovery.                                                                                                                                                                 |
| **GRUB_ENABLE_CRYPTODISK**     | This will make all commands look for encrypted disks and add the commands needed to access them during boot. This disables automatic booting because a passphrase is needed to decrypt the disks before they can be accessed. |

## Managing Menu Entries

When `update-grub` is run, GRUB 2 will scan for kernels and operating systems on the machine and generate the corresponding menu entries on the `/boot/grub/grub.cfg` file. New entries can be manually added to the script files with the `/etc/grub.d` directory.

| Paratmeters            | Purpose                                                                                       |
| ---------------------- | --------------------------------------------------------------------------------------------- |
| `set root`             | define the disk and partition where the root file system for the operating system is located. |
| `--no-floppy`          | GRUB will ignore floppy disks during a search operation.                                      |
| `--set=root --fs-uuid` | This will setup the root filesystem according to the UUID.                                    |
| `linux`                | This line indicates where the kernel for the operating system is located (/vmlinuz).          |
| `ro`                   | read only                                                                                     |
| `quiet`                | Disable log messages during startup                                                           |
| `splash`               | A splash screen should be shown during startup                                                |
| `initrd`               | Indicates where the initial RAM disk is located.                                              |

> [!NOTE]
> Disks are numbered from zero, so hd0 is first disk.
> Partitions are numbered from one, so `(,1)`, or `sda1`.

## Booting from the GRUB 2 shell

You can use the GRUB 2 shell to boot the system in case a misconfiguration in a menu entry causes it to fail.

1. Find out where the boot partition is.

```bash
ls
```

Output:

```block
(proc) (hd0) (hd0,msdos1)
```

The firs partition of `hd0` is called `msdos1` because the disk was partitioned using the [[MBR Partition]]  scheme. If it were partitioned using [[GPT Partition]], the name would be `gpt1`.

2. Check the content of the partition

```bash
ls (hd0,msdos1)/
```

Output:

```block
lost+found/ swapfile etc/ media/ bin/ boot/ dev/ home/ lib/ lib64/ mnt/ opt/ proc/ root/
run/ sbin/ srv/ sys/ tmp/ usr/ var/ initrd.img initrd.img.old vmlinuz cdrom/
```

3. Set the boot partition

```bash
set root=(hd0,msdos1)
```

4. Load the Linux kernel with the linux command, followed by the path to the kernel and the `root=` option to tell the kernel where the root filesystem for the operating system is located.

```bash
linux /vmlinuz root=/dev/sda1
```

5. Load the initial RAM disk with `initrd`, followed by the full path to the `initrd.img` file

```bash
initrd /initrd.img
```

Now, boot the system with `boot`.


## Boot from the Rescue Shell

The process to boot a system from this shell is almost the same as shown before.

After finding out which partition is the boot partition with `ls`

```bash
set prefix=(hd0,msdos1)/boot/grub
```

Now load the modules

```bash
insmod normal
insmod linux
```

Next, continue like the steps before and try to boot with `boot`.
